# Weekly AI & Tooling Digest — 2026-W18

*Week of 2026-04-20 to 2026-04-26*

## 🔴 Critical

### [Claude Code v2.1.117–119: MCP Tool Hooks, Custom Themes, Multi-Platform PR Support](https://github.com/anthropics/claude-code/releases)

Three releases shipped April 22–23. Headlines: hooks can now invoke MCP tools directly (`type: "mcp_tool"`); custom themes are creatable via `/theme` and shippable inside plugin `themes/` directories; `/cost` and `/stats` merged into a single `/usage` command; `--from-pr` now accepts GitLab, Bitbucket, and GitHub Enterprise URLs; `/config` settings persist to `~/.claude/settings.json` with project/local/policy override precedence; and hooks now include a `duration_ms` field for tool execution timing.

**Why it matters:** The `type: "mcp_tool"` hook type is a meaningful agentic primitive — Claude Code's event stream can now directly trigger external MCP tools mid-session without manual CLI glue. The `prUrlTemplate` setting and multi-platform `--from-pr` are useful if you ever review code against non-GitHub remotes.

### [Claude Agent SDK Python v0.1.65–0.1.68: SessionStore Batch API, Thinking Control, Trio Fix](https://github.com/anthropics/claude-agent-sdk-python/releases)

v0.1.65 (April 23) added `SessionStore.list_session_summaries()`, `import_session_to_store()`, and a `display` field in `ThinkingConfig` for controlling extended-thinking visibility. v0.1.67 (April 25) patched a `RuntimeError: no running event loop` regression that broke `ClaudeSDKClient` under Trio, now using sniffio-based dispatch.

**Why it matters:** If you're building any agentic pipelines with the Claude Agent SDK, the Trio fix prevents silent async breakages; `SessionStore.list_session_summaries()` is the right primitive for building session-history UIs or digest tooling on top of Claude.

### [Cloudflare Agents Week 2026: Dynamic Workers, Sandboxes GA, Workflows v2, Flagship Feature Flags](https://blog.cloudflare.com/agents-week-in-review/)

Agents Week (April 13–20) delivered 20+ product launches. The most infrastructure-relevant: **Dynamic Workers** — isolate-based sandboxes for AI-generated JavaScript that spin up in milliseconds and scale to millions of concurrent executions at a fraction of container cost. **Sandboxes GA** gives agents persistent shell/filesystem environments. **Workflows v2** now supports 50,000 concurrent workflows. **Flagship** is a native feature-flag service built on KV + Durable Objects with sub-millisecond evaluation latency. **AI Gateway** expanded to 14+ model providers with a direct Workers binding.

**Why it matters:** Dynamic Workers and Workflows v2 are direct upgrades to what you can build on Cloudflare — running agent-generated code at scale without containers and orchestrating multi-step flows. Flagship is a drop-in feature flag layer for TypeScript/Workers apps that requires no third-party service, which is relevant to Bucktee feature rollouts.

### [Codex CLI v0.125.0: Unix Socket Transport, Reasoning-Token Reporting, GPT-5.5](https://github.com/openai/codex/releases)

Stable 0.125.0 (April 24) adds Unix socket transport for app-server integrations, permission profiles that now persist across TUI sessions and MCP sandbox states, and `codex exec --json` reporting reasoning-token usage. GPT-5.5 is accessible via the Codex model selector in this build; an external writeup documented API access via the Codex backdoor route (simonwillison.net, April 23). v0.126.0-alpha.3 dropped April 26.

**Why it matters:** Reasoning-token reporting in `codex exec --json` enables apples-to-apples cost comparison with Claude Code when evaluating which CLI agent to use for a given task type.

v0.39.1 (April 24) is the stable patch for the v0.39 series. The v0.40.0 previews (April 23–25) bundle offline ripgrep, improved skill extraction, and the upcoming memory-system refactor that replaces `MemoryManagerAgent` with prompt-driven memory editing across four tiers.

### [GitHub Copilot Plans Restructured; New Pro Signups Paused](https://github.blog/changelog/month/04-2026/)

On April 20, GitHub split Copilot Individual into essential and professional tiers and temporarily paused new signups for Pro, Pro+, and Student plans. The stated reason: agentic workflows are consuming far more compute than the original pricing model assumed, with single requests sometimes exceeding the cost of an entire monthly plan. Claude Opus 4.7 is also rolling out on the Copilot platform this week.

**Why it matters:** This signals a broader repricing of agentic compute that will hit every provider. Opus 4.7 on Copilot is notable because it's the same model that powers Claude Code's most capable reasoning — if you use both tools, expect some capability convergence.

### [WooCommerce Product Filter by WBW — Unauthenticated Config Deletion (CVE-2026-3138)](https://freshysites.com/security-bulletins/wordpress-security-bulletin-product-filter-for-woocommerce-by-wbw-plugin-vulnerability-cve-2026-3138/)

**Why it matters:** If Bucktee's environment or any WooCommerce site you maintain uses the WBW Product Filter plugin, this is an immediate patch — unauthenticated write/delete access to plugin state is a high-severity issue regardless of CVSS score.

## 🧠 Takeaways

- **MCP tool hooks are Claude Code's sleeper feature this week.** `type: "mcp_tool"` in hooks turns Claude Code's event system into a lightweight automation bus — you can now pipe tool-use events directly to external services without writing wrapper scripts. Start experimenting with this before everyone discovers it.
- **Cloudflare just repositioned itself as an AI-native runtime, not just a CDN.** Dynamic Workers + Sandboxes GA + Workflows v2 in one week is a coherent platform bet: Cloudflare wants to be the execution layer *under* your agents. If you're building anything agentic on Workers, the primitives just got significantly more powerful.
- **Agentic compute economics are breaking legacy pricing models.** Copilot's signup pause and GitHub's public admission that single agentic requests can exceed a monthly plan's cost are early signs. Expect repricing across providers in Q2/Q3 2026 — Claude Code's Pro plan caps will likely shift too.
- **Migrate off `claude-haiku-3` now if you haven't.** It was retired April 19. Any hardcoded `claude-haiku-3` references in Bucktee, automation scripts, or plugin code will fail silently — replace with `claude-haiku-4-5-20251001`.
