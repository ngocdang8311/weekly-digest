# Weekly AI & Tooling Digest — 2026-W19

*Week of 2026-04-27 to 2026-05-03*

## 🔴 Critical

### [Claude Code v2.1.126: Project Purge, Gateway Model Picker, OAuth-in-Terminal, Security Fix](https://github.com/anthropics/claude-code/releases)

Shipped May 1. Four headline changes: (1) `claude project purge [path]` deletes all Claude Code project state — transcripts, tasks, file history, config entry — with `--dry-run`, `--all`, and interactive modes; (2) the `/model` picker now queries your gateway's `/v1/models` endpoint when `ANTHROPIC_BASE_URL` points to an Anthropic-compatible proxy; (3) `claude auth login` now accepts OAuth codes pasted directly into the terminal when the browser callback can't reach localhost — critical for WSL2, SSH sessions, and containers; (4) **security fix**: `allowManagedDomainsOnly` and `allowManagedReadPathsOnly` config flags were silently being ignored and are now enforced. Also fixed: stream idle timeouts after Mac sleep, CJK text garbling on Windows, and large image pastes breaking sessions.

**Why it matters:** `claude project purge` solves real context-hygiene problems when switching between Bucktee plugin work and other client projects. The OAuth terminal-paste fix is the last major blocker for running Claude Code inside Docker or remote SSH — no more manual workarounds.

### [Codex CLI v0.128.0: Persisted /goal Workflows, Self-Update, Configurable Permission Profiles](https://github.com/openai/codex/releases)

Stable release April 30. Key additions: persisted `/goal` workflows with app-server APIs, TUI controls for create/pause/resume/clear, and runtime continuation across sessions; `codex update` self-update command; permission profiles are now configurable with built-in sandbox presets and persist across TUI sessions, MCP sandbox state, and shell escalation. Plugin workflows gained marketplace installation, remote bundle caching, and plugin-bundled hooks. `codex exec --json` now reports reasoning-token usage.

**Why it matters:** Persisted `/goal` is a direct parallel to Claude Code's project mode — worth evaluating side-by-side if you run long Cloudflare Workers refactors or Bucktee release cycles across multiple sessions. Reasoning-token reporting keeps cross-tool cost comparisons honest.

### [Gemini CLI v0.41.0-preview.0: Real-Time Voice Mode, Gemma 4, Shell Validation Hardening](https://github.com/google-gemini/gemini-cli/releases)

Preview dropped April 28; v0.40.1 stable patch April 30. Preview highlights: real-time voice mode with cloud and local backends; Gemma 4 model support (experimental); enhanced shell command validation via a core tools allowlist; async experiment/quota fetching for faster startup; secure `.env` loading in headless mode. v0.40.1 stable fixes transient error marking to prevent retryable failures from being treated as terminal.

**Why it matters:** Voice mode in a terminal CLI is something neither Claude Code nor Codex CLI has shipped yet — Gemini CLI is moving faster on input modalities. The multi-model strategy (Gemini + Gemma 4) also previews where CLI tooling is heading: model-agnostic, user-selectable backends.

### [Cloudflare: @cloudflare/dynamic-workflows — Durable Execution Inside Dynamic Workers](https://developers.cloudflare.com/changelog/post/2026-05-01-dynamic-workflows/)

Shipped May 1. Enables attaching Cloudflare Workflows to Dynamic Workers — code loaded at runtime rather than deployed statically. The core problem: when a Workflow sleeps between steps, the Dynamic Worker that initiated it may no longer be in memory at resume time, losing state. The library solves this by tagging each Workflow instance with metadata identifying which Dynamic Worker to reload on resume, ensuring durability across multi-hour or multi-day sleep gaps.

**Why it matters:** If you're building agentic pipelines on Workers — e.g., background jobs that call AI APIs, write to KV, sleep, then continue — this finally makes them truly durable without forcing a static deployment model. Pairs directly with Workflows v2's 50k concurrent workflow support from last week.

## 🟡 Noteworthy

### [GitHub Copilot Moving to Usage-Based Billing June 1](https://github.com/orgs/community/discussions/192963)

All GitHub Copilot plans transition to AI Credits billing on June 1, 2026. Every plan includes a monthly allotment; overages are billed per token (input, output, cached). A preview billing dashboard is rolling out in early May so users and admins can see projected costs before the cutover. Heavy agentic use — Copilot Workspace, Copilot CLI, multi-file edits — consumes significantly more credits than inline autocomplete.

**Why it matters:** If you run Copilot alongside Claude Code for any agentic workflows, review your projected usage before June 1 — the old flat-rate assumption is gone, and a single complex agent session can now show up as a line item.

### [GitHub Copilot CLI Reaches General Availability; v1.0.40 Adds MCP OAuth](https://www.infoq.com/news/2026/04/github-copilot-cli-ga/)

Copilot CLI is now GA with natural language terminal commands and code explanation. v1.0.40 (May 1) adds MCP server support using `client_credentials` OAuth grant type, smoother streaming output, and `/clear` and `/new` correctly resetting active custom agent selection.

**Why it matters:** MCP support in Copilot CLI puts it in direct competition with Claude Code's MCP server ecosystem. If you've built or rely on custom MCP servers, they may now be usable from multiple CLI agents — worth testing for portability.

## 🧠 Takeaways

- **Claude Code v2.1.126's security fix is easy to miss but important.** `allowManagedDomainsOnly` and `allowManagedReadPathsOnly` being silently ignored was a latent policy-bypass bug. If you've configured domain or path restrictions for team use, verify your settings actually take effect now and weren't previously ineffective.
- **Cloudflare's agent platform is snapping together.** Dynamic Workflows + last week's Workflows v2 + Sandboxes GA form a coherent execution stack for long-running AI agents. If you've been waiting for Cloudflare Workers to be a credible backend for multi-step agentic work, the stack is now mature enough to prototype seriously.
- **Gemini CLI is the velocity leader on new input modalities.** Voice mode, Gemma 4 support, and offline ripgrep bundling shipped in a single preview — Google is treating the CLI as a first-class product, not a wrapper. Claude Code's terminal-first philosophy is increasingly being matched, not differentiated.
- **The June 1 Copilot billing flip is the most consequential pricing event of the quarter.** Token-based billing for agentic workflows will shock teams that treat Copilot as a fixed cost. If you manage any shared org accounts, pull the usage preview now before the surprise hits invoices.
